## Key Learnings

### 1. Recognition â‰  Establishment

**Discovery**: Perplexity recognized the entity on Day 17 but lost recognition by Day 19.

**Implication**: 
- Initial recognition is a milestone, not the goal
- True establishment requires sustained recognition (7-14+ days)
- Stability testing must be part of methodology
- Single successful query doesn't confirm permanent entity

**Recommendation**: 
- Extend stability verification to 7-14 days minimum
- Test multiple query variations
- Document stability patterns per platform
- Don't declare success until stability confirmed

---

### 2. Platform Characteristics Vary Dramatically

**Discovery**: ChatGPT (9 days, stable) vs. Perplexity (17 days, volatile) vs. Claude (20+ days, not achieved)

**Implication**:
- One methodology doesn't produce uniform results
- Each platform has different indexing cycles
- Stability patterns are platform-specific
- Timeline predictions must be platform-adjusted

**Recommendation**:
- Test each platform separately with extended timelines
- ChatGPT: 7-14 days to recognition, stable immediately
- Perplexity: 14-21 days to recognition, 30+ days for stability
- Claude: 21-30 days to recognition, stability TBD
- Don't expect cross-platform consistency

---

### 3. Early Recognition Correlates with Stability

**Discovery**: ChatGPT recognized Day 9 (38% into window) = stable. Perplexity recognized Day 17 (100% into window) = unstable.

**Hypothesis**: Earlier recognition within predicted window suggests stronger entity signal and better long-term stability.

**Implication**:
- Late-window recognition may be weaker
- More external sources may accelerate recognition
- Stronger initial signal = better stability

**Recommendation**:
- Front-load external content (publish Day 1-3, not Day 10)
- Aim for early-window recognition (30-50% into window)
- If late-window recognition occurs, expect volatility
- Double reinforcement period for late recognition

---

### 4. Structured Data is Necessary But Not Sufficient

**Discovery**: All platforms have same structured data access, but results differ dramatically.

**Implication**:
- JSON-LD schemas are processed differently per platform
- Structured data alone doesn't guarantee recognition
- Other factors (time, sources, authority) matter significantly
- Schema.org is foundation, not complete solution

**Recommendation**:
- Implement comprehensive structured data (baseline requirement)
- Combine with multi-source external validation
- Allow sufficient time (20-30 days minimum)
- Don't rely on schemas alone for recognition

---

### 5. Query Sensitivity Matters

**Discovery**: Perplexity Day 17 (recognized) vs. Day 19 (not recognized) with identical query.

**Implication**:
- Exact query phrasing affects results
- Entity-first queries ("NoorRank Labs' protocol") may work better than generic ("Miracle Berry Protocol")
- Disambiguation remains challenging for ambiguous terms
- Success measurement should test multiple query angles

**Recommendation**:
- Test 5-7 query variations per platform
- Include entity-first queries
- Include contextual queries (broader topics)
- Document which queries work best
- Optimize for query variations, not just entity name
